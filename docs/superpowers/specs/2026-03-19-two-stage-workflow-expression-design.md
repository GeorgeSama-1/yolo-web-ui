# Two-Stage Workflow Expression Design

## Goal

让页面一眼就能表达出两阶段工作流：

- 检测模型
- 产出绝缘子框
- 分类模型
- 产出 normal / abnormal

并且在未配置分类模型时，仍然保持一阶段流程可理解。

## Problem

当前页面虽然已经能显示一阶段数量和二阶段统计，但“两个模型如何串起来工作”主要靠用户自己推断。右侧更像状态卡片集合，而不是流程链路。

## Approaches

### 方案 1：说明文字

只在中间或右侧补一段说明文字。

- 优点：改动最小
- 缺点：用户需要自己阅读和翻译，不够直观

### 方案 2：可视化流程链路

在右侧任务面板顶部增加固定的流程链路：

- 检测模型
- 绝缘子框
- 分类模型
- normal / abnormal

每个节点显示当前状态、模型名或结果计数。

- 优点：工作流一眼可懂
- 缺点：需要重排右侧顶部结构

### 方案 3：重流程图/HUD

加入更复杂的线路图、状态灯和连线。

- 优点：科技感更强
- 缺点：容易过重

## Chosen Design

采用方案 2。

### Right Sidebar Structure

右侧顶部新增 `流程链路` 区块，优先于结果统计。

链路固定为四个节点：

1. 检测模型
2. 绝缘子框
3. 分类模型
4. normal / abnormal

### Node Semantics

#### 检测模型

- 显示当前检测模型名
- 如果模型在线，显示 active 状态

#### 绝缘子框

- 显示一阶段数量
- 表达这是检测模型的直接产物

#### 分类模型

- 已配置时显示模型名和 active 状态
- 未配置时显示未启用

#### normal / abnormal

- 已启用两阶段时显示 normal / abnormal 数
- 未启用时显示等待分类

### Interaction Expectations

- 用户首先在右侧看懂系统工作顺序。
- 中间工作区继续负责表达当前图片的阶段归属。
- 右侧回答“系统如何处理”，中间回答“这张图得到了什么结果”。

## Testing

通过源码级测试锁定：

- `流程链路`
- `检测模型`
- `绝缘子框`
- `分类模型`
- `normal / abnormal`
- 未启用分类时的降级文案
