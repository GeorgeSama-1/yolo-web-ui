# Web UI Stage Bus Design

## Goal

把中间工作区顶部改成更像主工作台的“阶段总线”，让用户一眼看懂哪些结果来自一阶段检测，哪些结果来自二阶段分类。

## Problem

当前中间区域虽然已经显示了一阶段数量、二阶段 normal/abnormal 和当前选中，但几块卡片权重接近，用户需要自己阅读文案才能理解结果归属。工具条也更像一排按钮，而不是工作台的控制总线。

## Approaches

### 方案 1：只改文案

只把现有卡片标题改得更明确，例如“一阶段检测数量”“二阶段异常数量”。

- 优点：改动最小
- 缺点：信息层级还是平的，仍然需要用户自己解释流程

### 方案 2：阶段总线

把工具条下面的状态区拆成三个有明确顺序的阶段块：

- 阶段 1 检测
- 阶段 2 分类
- 当前选中

每块都有自己的颜色语义、标题和关键数值。

- 优点：结果归属最清楚
- 缺点：需要重排中间顶部结构

### 方案 3：重 HUD 仪表盘

进一步加入复杂状态灯和装饰连线。

- 优点：科技感最强
- 缺点：容易喧宾夺主

## Chosen Design

采用方案 2。

### Top Toolbar

顶部工具条只承担“操作”职责：

- 缩放
- 适应窗口
- 1:1
- 补框
- 当前图片名

工具条视觉统一为深色科技工具栏，不在这里混入结果统计。

### Stage Bus

工具条下方改成“结果归属条”，顺序固定为：

1. 阶段 1 检测
2. 阶段 2 分类
3. 当前选中

阶段 1 使用蓝青色，显示：

- 总绝缘子数
- 类别计数摘要

阶段 2 使用紫红色，显示：

- normal
- abnormal
- 若未启用分类模型，明确显示“未启用分类”

当前选中使用琥珀色，显示：

- 当前选中实例数
- 单选时的实例编号提示

### Interaction Expectations

- 用户应能直接按从左到右的顺序读懂当前结果来源。
- 未启用分类模型时，二阶段区域仍然存在，但显示为未启用状态，而不是完全消失。
- 删除选中等针对当前实例的动作，保持靠近当前选中区域。

## Testing

使用源码级测试锁定中间工作区关键文案，至少覆盖：

- 流程总线
- 阶段 1 检测
- 阶段 2 分类
- 未启用分类
- 当前选中
